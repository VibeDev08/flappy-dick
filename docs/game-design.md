# Game Design

## Pillars

- fast restarts
- readable obstacles
- crude but punchy presentation
- mobile-first portrait feel
- enough variety to stay funny without changing the rules every run

## Player Avatar

The player avatar is a double-ended dildo with 2 launch variants:

- a smaller lighter-skin variant
- a slightly larger darker-skin variant

Both play the same. The visual difference is cosmetic and does not create separate difficulty classes.

## Obstacles

Obstacles replace pipe tunnels with penis-shaped blockers.

Rules:

- shaft and head only
- no balls
- no body hair
- must read clearly at gameplay speed
- obstacle variety is mostly visual, not mechanical
- collision is a little forgiving

Launch target:

- all obstacles use one clear shared penis silhouette
- gameplay variety comes from spacing, gap placement, and arrangement
- every obstacle must visibly include a shaft and readable head shape

## Core Loop

1. `18+` confirmation
2. character selection
3. immediate run start
4. flap through obstacles and score points
5. die on collision or leaving bounds
6. show goo splatter fail state
7. retry quickly or enter name if top 10 qualifies

## Fail Screen

- a white goo splatter appears on death
- score text sits inside the splatter
- splatter size scales with score
- splatter growth clamps to available viewport size
- retry stays nearly instant

## Copy Style

- crude puns
- short lines
- readable score-first layout

## Audio

- minimal-funny sound effects
- one looping background track
- no audio clutter

## Share Card

The share card includes:

- score
- selected character
- a funny line
- the game URL

## Leaderboard

- all-time only
- top 10 only
- prompt for name only if the run qualifies
- basic abuse filtering
- moderate anti-cheat using signed run tokens
